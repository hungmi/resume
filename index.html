主要成果

    Zuker：招租網站暨租賃管理系統

    台北的包租代管公司。我們想盡辦法用越少人管理越多房間以達到最大利潤。因此從創業初期就自行開發系統。直到現在，以十人的團隊管理超過兩百間房，同時經營超過十萬訂閱的 YouTube 頻道。

    • 與各部門共同釐清招租、簽約及管理問題，轉化為功能規劃，安排合適的開發順序，逐步交付適當的功能協助整間公司遠端協作。
    • 曾導入外包團隊，後續由我改善其程式效能問題及架構：補充相關測試、建立快取及優化 slow queries。
    • 權限採用 Pundit 進行管理，分別為房客、房東、房東助手提供不同的功能。
    • 從洽詢房屋開始，房客可在網站上完成所有手續，包含入住後的繳交租金，退租時的押金退還。
    • 初始專案為 Rails 4.1，目前已升級至 Rails 6.1，並搭配 Turbo (相關升級筆記) & stimulus.js & esbuild 架構出最適合小團隊進行開發之架構。
    • 測試採用 Minitest & Capybara，範圍包含網站所有主要流程。
    • 全站使用 Rails i18n 標準架構製作雙語，目前支援中英文。
    • AppWorks 加速器第 12 屆。

    Polish Design：磨人創意網路有限公司

    由於我想維持遠端工作，我嘗試了與前端夥伴、設計夥伴一同成立接案公司。為了達到客戶期待，我不只負責所有後端工程、伺服器部署，也主導多個網站的功能規劃，並導入自動化測試，及協助部分前端工程。

    • 重點案例：
      1. 遊戲橘子集團 絕對武力 官方網站：負責專案管理、後端 API 協調及部分前端工程。
        • 此網站我負責與遊戲橘子內部 MIS 協作制定後台操作邏輯及後端 Api，並設定 Postman mock server 方便夥伴一同開發。
        • 工程部分我與前端夥伴分頭開發再整合。
      2. 四合願 時間拖延器 上市暖身網站：
        • 此網站為一款小遊戲，整體操作過程較多變化，因此我透過 GTM 客製化搜集數據，方便客戶行銷部門透過數字理解使用者投入時間、遊戲過程數據。
        • 除此之外我負責後端工程及部署。
      3. Wassup 貓砂 購物網站：
        • 此網站我將較複雜的前端拆開與夥伴以 Vue.js 串接 Api 製作，其餘部分我以 Rails 直接 SSR 渲染，取得開發速度的平衡。自動化測試包含全站購物流程（Vue.js 的部分以瀏覽器自動測試）。
        • 在購物功能上相別一般電商，另有提供可彈性發貨的不固定頻率訂閱方案。
